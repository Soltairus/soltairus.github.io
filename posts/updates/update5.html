Update 5: Multiple Enemy Combat

November 7, 2017.

Welcome back to Project Phases. This is Update 5 which is part 2 of multiple player combat which covers the enemy side of things. Not much of an introduction, but we got a lot to cover so let’s start!

Main Menu Changes

If you make your way to the Player Select menu, you will see 3 new people on the bottom. These are the enemies which you can select and unselect just like the players. The enemies are Clockman, Crocman and Lockman. I got the ideas for the characters from a drawing my friend made, so I technically didn’t make them. Converting the drawing into 16x16 sprites was fun and I think they all turned out great. My favorite is Lockman just because how well his suit complements the rest of his design. If you want a closer look at the enemies, go to the Project Phases tab on the website and scroll to the “Enemy” section.


Battle Changes

This is where the bulk of the update is. Now when you go into a battle, you will see all three enemies on the right. I think it looks great with all six characters on each side. It makes everything look more complete.


The first big change is the new selection system. I revamped the system from the last update that I used for the Choice Heal attack and applied it to all the moves. Now, once you select an attack you can select a target. The future arrow will move accordingly to where it will attack. The future arrow is still really buggy and doesn’t work all the time, so I hope to get that fixed soon. Also, Oliver’s Wide Slash attack now works since we have all the enemies. You don’t have to select anyone to use Wide Slash.

The Enemy AI as received a huge overall. Before, it was based on one player’s state in the timeline but now it is based on a priority system. Every enemy goes through a list of priorities, whenever one is true it uses it. Here is that list…

-Lowest Health
-Max Health
-Defend Phase
-Random
-Max Point

Some enemies have it ordered differently so not every enemy is attack the same person. For example, one enemy might check who has the lowest health. If that person is in the defend phase, they will try checking who has the most points, then attack. If that person is also in the defend phase, they might just go ahead and attack or randomly pick someone. He may also heal or wait depending on what his health is. If everyone is in the defend phase, given the enemy has enough points, it may just skip. There are many options, but there is still a lot of work to be done. In the future, I hope to add some way where you can figure out what each enemy’s priorities are so you can build strategies around it.

If you want to see a full battle, at 11:36 in the update video I demonstrate a battle and all of the thought that can go behind it. The video should be embedded above, so go check it out. I also talk about some future mechanics in that video. But it doesn’t last long since the game crashed, but you get the point.

Overall, Update 4 and 5 combined has been the biggest update yet. It’s really helped the game come together. Next update will be additions to the overworld since we have been working in the battle system for a long time.

Until next time.

Thanks for reading,
